Local AI Hiring Demo
Enterprise AI-Driven Technical Assignment Generator

Project Overview

This project is a full-stack, enterprise-level AI hiring automation system. It automates candidate screening, role-specific technical assignment generation, PDF documentation, and email delivery using Node.js, Express, OpenRouter AI, and more.

Folder Structure

  • webhook-server/ – Main backend server (Node.js, Express, AI, PDF, Email)
  • github-demo/ – GitHub repo creation logic (mock/demo)
  • openclaw-skills/ – OpenClaw skills and integration (future scope)
  • audit-logs/ – Candidate logs (raw and processed JSON)
  • demo-form/ – Frontend HTML demo form for local testing

How It Works

  1. Candidate applies via the HTML form, uploading their resume PDF and filling in details.
  2. API POST /api/career-apply receives the form (with multer for file upload).
  3. Resume PDF is parsed using pdf-parse to extract project, tech, and domain keywords.
  4. OpenRouter AI is called with a role-specific, resume-personalized prompt to generate a unique, enterprise-level technical assignment (JSON structure).
  5. PDFKit generates a multi-section, professional assignment PDF (2-3 pages) with all assignment details.
  6. Nodemailer sends the PDF to the candidate's email address.
  7. Audit logs are updated with both raw and processed candidate data.

API Endpoint

POST /api/career-apply
Form fields: fullName, email, appliedRole, yearsExperience, skills, resumeFile (PDF)
  • Receives candidate info and resume PDF.
  • Returns { success: true, assignment, hrScore } on success.

Assignment Generation Logic

  • Role-specific: Assignment is different for UI/UX, Frontend, Backend, Full Stack, Python, QA, DevOps, etc.
  • Resume-aware: AI prompt uses extracted resume keywords (e.g., healthcare, ecommerce, CRM) to personalize the project domain.
  • AI Model: Uses OpenRouter google/gemma-3-27b-it:free for assignment generation.
  • Fallback: If AI fails, a smart fallback assignment is generated for the role.

PDF Assignment Document

  • Company header, candidate details, HR score
  • Project title, introduction, business objective
  • Functional modules, technical requirements
  • Deliverables, evaluation criteria, timeline, stack
  • Final HR notes
PDF is 2-3 pages, formatted for professional company use.

How to Run

  1. Clone/download the project.
  2. Install dependencies:
    cd webhook-server && npm install
  3. Copy .env.example to .env and fill in your Gmail/SMTP and OpenRouter API key.
  4. Start the server:
    npm start
  5. Open demo-form/demo-form.html in your browser and submit a test application.

Notes

  • This is an enterprise-level demo. For production, secure your API keys and validate all inputs.
  • OpenClaw integration is for future scope.
  • GitHub repo creation is mocked/demo only.
